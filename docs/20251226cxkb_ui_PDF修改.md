# 知识库 PDF 列表 UI 优化方案

## 1. 背景
当前知识库（Knowledge Base）中的 PDF 列表项采用单行 Flex 布局，导致 PDF 名称、文件大小以及操作按钮（选择、预览、重命名、删除）全部挤在同一行。在小屏幕或名称较长时，排版显得非常拥挤，用户体验不佳。

## 2. 目标
优化 PDF 列表项的排版，将其改为两行显示：
- **第一行**：显示 PDF 的名称（DisplayName）和文件大小。
- **第二行**：显示 PDF 的操作按钮（选择、预览、重命名、删除）。

## 3. 修改方案

### 3.1 目标文件
- `99AI-ANKI/chat/src/views/chat/components/sider/KnowledgeBaseReadonly.vue`

### 3.2 结构调整
将原有的水平 Flex 布局调整为复合布局：
1.  **外层容器**：保持 `flex` 布局，但对齐方式改为 `items-start`（顶部对齐），以适应多行内容。
2.  **左侧图标**：保持原样，作为固定宽度的装饰。
3.  **右侧内容区**：改为 `flex flex-col`（垂直布局），包含两行：
    - **第一行 (Header)**：
        - 左侧：PDF 名称（`truncate` 截断）。
        - 右侧：文件大小（`shrink-0` 不缩放）。
    - **第二行 (Actions)**：
        - 放置所有的操作按钮（选择、预览、重命名、删除）。
        - 使用 `flex items-center gap-4` 增加按钮间距，提升点击精度。

### 3.3 样式优化
- 增加行间距（`gap-1` 或 `gap-2`）。
- 调整按钮样式，使其在第二行排列整齐。
- 确保在选中状态（`selectedKbPdfId === f.id`）下的高亮效果依然正确。

## 4. 详细修改步骤

1.  定位到 `KnowledgeBaseReadonly.vue` 的 `<template>` 部分，找到渲染 PDF 列表的 `v-for` 循环。
2.  修改列表项的 `div` 容器类名。
3.  重构内部的 `div` 结构，将名称和按钮分层。
4.  调整按钮的间距和字体大小，确保在移动端也有良好的表现。

## 5. 预期效果
- PDF 名称拥有更大的显示空间。
- 操作按钮独立成行，更加醒目且易于点击。
- 整体视觉效果更加规整，不再拥挤。
